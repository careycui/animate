近期使用的动画效果

  • 2D平面变换效果

  • 主要是渐变、位移的变换，称之为过渡效果更合适。从一个状态过渡到另一个状态，实现方法简单：javascript模拟帧、css3实现。模拟帧利用javascript定时器实现，CSS3利用新特性实现。

    所有的过渡效果在2维平面内进行，在平面内完成位移、透明度、字体大小等CSS数值属性的变换。 渐变：opacity的改变；位移：x/y轴的坐标移动

  • 3D立体空间变换效果

  • 以假想的三维空间为舞台，涉及到x，y，z及视点的变化。 简而言之就是要在浏览器平面上表现出近大远小、多平面的效果。

动画帧请求回调函数列表

不需要使用者指定循环间隔时间，浏览器会基于当前页面是否可见、CPU的负荷情况等来自行决定最佳的帧速率，从而更合理地使用CPU。

特性： A． 会集中每帧的DOM操作，在一次reflow中完成全部重绘，reflow的时间依据浏览器的刷新频率而定。减少了重绘次数，集中操作，解决了不同浏览器的刷新频率不同的问题。 B． 隐藏、不可见元素，不进行重绘。（更少内存，GPU的使用）

全屏滚动动画一些总结

  • 使用同一个元素控制动画
  • 复杂动画可用嵌套的多个元素表示
  • 提取公用动画
  • 避免transform变换冲突
  • 采用居中定位原则
  • 不使用动画关键帧定位元素
类控制
element{/* 样式 */}
.active element{ [/* 最终定位 */] /* 动画元素 */}
animation-play-state控制
element{/* 样式 */ /* 动画元素 */}
.animate{ animation-play-state: paused }
.active element{ [/* 最终定位 */] animation-play-state: running }

为了使动画模块化，便于管理扩展

一个场景：滚动到每屏时，页面动画元素A从左侧旋转淡入，页面元素B重底部旋转淡入等等， 每个元素都具有‘淡入’的效果。

DOM a 具有旋转（0-360deg）、放大（0-100%）的动画效果。这两个动画效果具有通性，可能 在最初已被提取为公共动画（rotate1、scale1）。当在为元素a应用两个动画时 a {animation:rotate1 1s,scale1 1s}旋转效果会被残忍的覆盖掉。

以中心点为参考点定位元素。主要处理不同屏幕尺寸的显示问题以及可以统一管理。

谨慎使用animation-fill-mode（可以以动画帧决定元素开始结束位置）
